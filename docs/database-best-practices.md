# Database Best Practices

PostgreSQL for production, SQLite for lightweight/local projects. SQLAlchemy as the sole ORM.

## Mandatory Tools

| Tool | Purpose | Install |
|------|---------|---------|
| `sqlalchemy` | ORM + core query builder | `uv add sqlalchemy` |
| `alembic` | Schema migrations | `uv add alembic` |
| `asyncpg` | Async PostgreSQL driver | `uv add asyncpg` |
| `aiosqlite` | Async SQLite driver | `uv add aiosqlite` |
| `pgcli` | Interactive PostgreSQL client | `uv tool install pgcli` |

## Quality Gates

| Gate | Command | Blocks on |
|------|---------|-----------|
| Pending migrations | `alembic check` | Unapplied migrations |
| Migration lint | Review autogenerated migration before commit | Destructive ops without confirmation |
| Rollback test | `alembic upgrade head && alembic downgrade -1 && alembic upgrade head` | Rollback failure |
| No N+1 | Enable `echo=True` or SQLAlchemy warn-on-lazy-load in tests | Lazy load detected |

## SQLAlchemy Patterns

### Models — Mapped Classes

```python
from __future__ import annotations
from datetime import datetime
from uuid import uuid4
from sqlalchemy import String, Text, ForeignKey, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship

class Base(DeclarativeBase):
    pass

class User(Base):
    __tablename__ = "users"

    id: Mapped[str] = mapped_column(primary_key=True, default=lambda: str(uuid4()))
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True)
    name: Mapped[str] = mapped_column(String(100))
    created_at: Mapped[datetime] = mapped_column(server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(server_default=func.now(), onupdate=func.now())

    posts: Mapped[list[Post]] = relationship(back_populates="author", lazy="raise")
```

### Key Rules

- **`lazy="raise"`** on all relationships — forces explicit loading, prevents N+1
- **`Mapped[]` type annotations** on every column — no `Column()` legacy style
- **`mapped_column()`** instead of `Column()`
- **UUID string PKs** for distributed-friendly IDs
- **`created_at` / `updated_at`** on every table via `server_default`

### Session Lifecycle (FastAPI)

```python
from collections.abc import AsyncGenerator
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine, async_sessionmaker

engine = create_async_engine(settings.database_url)
SessionLocal = async_sessionmaker(engine, expire_on_commit=False)

async def get_db() -> AsyncGenerator[AsyncSession, None]:
    async with SessionLocal() as session:
        yield session
```

- One session per request via `Depends(get_db)`
- `expire_on_commit=False` to avoid lazy loads after commit
- Never share sessions across requests or tasks

### Loading Strategies

```python
from sqlalchemy.orm import selectinload, joinedload

# One-to-many: selectinload (separate IN query)
stmt = select(User).options(selectinload(User.posts))

# Many-to-one: joinedload (single JOIN)
stmt = select(Post).options(joinedload(Post.author))

# Nested: chain them
stmt = select(User).options(selectinload(User.posts).joinedload(Post.tags))
```

- **`selectinload`** for collections (one-to-many) — avoids cartesian explosion
- **`joinedload`** for single references (many-to-one, one-to-one)
- Never rely on lazy loading — it's disabled via `lazy="raise"`

## Migration Discipline

```bash
alembic revision --autogenerate -m "add posts table"   # generate
# REVIEW the generated migration before committing
alembic upgrade head                                     # apply
alembic downgrade -1                                     # test rollback
```

- **Never edit a deployed migration** — create a new one to fix issues
- **Always review autogenerated code** — Alembic can't detect renames, data migrations, or index changes perfectly
- **One migration per logical change** — don't batch unrelated schema changes
- **Destructive ops** (drop column/table) require explicit confirmation from FX
- **Data migrations** go in separate migration files from schema migrations

## Query Optimization

- **Detect N+1**: Set `lazy="raise"` globally; enable SQL echo in tests
- **Indexing**: Index all FK columns, unique constraints, and frequent `WHERE`/`ORDER BY` columns
- **`EXPLAIN ANALYZE`**: Run on slow queries via `pgcli` before optimizing
- **Bulk operations**: Use `session.execute(insert(Model).values([...]))` for batch inserts — not loops
- **Connection pooling**: Configure `pool_size`, `max_overflow`, `pool_pre_ping=True` on engine
- **Pagination**: Always use keyset pagination (`WHERE id > :last_id`) over `OFFSET`

## Schema Design Conventions

| Convention | Rule |
|-----------|------|
| Table names | `snake_case`, plural (`users`, `blog_posts`) |
| Column names | `snake_case`, singular (`email`, `created_at`) |
| Foreign keys | `<singular_table>_id` (`user_id`, `post_id`) |
| Indexes | `ix_<table>_<column>` |
| Constraints | `uq_<table>_<column>`, `ck_<table>_<constraint>` |
| Timestamps | `created_at` + `updated_at` on every table |
| Soft deletes | `deleted_at: Mapped[datetime \| None]` — filter in repository layer |

## Testing with Real Databases

**Never mock the database** — use a real instance for integration tests.

```python
import pytest
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker

@pytest.fixture
async def db_session():
    engine = create_async_engine("sqlite+aiosqlite:///:memory:")
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    async with async_sessionmaker(engine)() as session:
        yield session
    await engine.dispose()
```

- **SQLite in-memory** for unit/integration tests — fast, disposable
- **PostgreSQL** for CI integration tests — catches dialect-specific issues
- **Transaction rollback** per test — each test starts clean
- **`factory-boy`** with `SQLAlchemyModelFactory` for test data
- **Assert on query count** to catch N+1 regressions in tests
