# Brainstorm — Interview Flow

Ask me one question at a time so we can develop a thorough, step-by-step spec for this idea. Each question should build on my previous answers, and our end goal is to have a detailed specification I can hand off to a developer. Let's do this iteratively and dig into every relevant detail. Remember, only one question at a time.

Aim for approximately 8 structured questions covering:
1. Problem space and target users
2. User personas and pain points
3. Success metrics and KPIs
4. Core capabilities and features
5. Scope boundaries (what we're NOT building)
6. Technical constraints and preferences
7. Priority and phasing
8. Acceptance criteria and definition of done

Adapt the questions dynamically based on answers received. Skip any topics already covered by `PROJECT-SEED.md` if it was detected.
